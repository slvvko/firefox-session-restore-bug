How to Test

  1. Open this page over HTTPS (Service Workers require HTTPS or localhost)
  2. Wait for the banner to show "SW active" (green) — the red ad boxes should disappear
  3. Test A — Ctrl+Shift+T:
    Close tab (Ctrl+W) → Restore (Ctrl+Shift+T)
  4. Test B — Full browser restart (more reliable):
    Quit the browser entirely → Reopen → Session restore activates the tab
  5. Check results:
    • Red ad boxes visible = SW was bypassed → the bug
    • Ads hidden, banner green = SW was consulted → correct behavior
  6. Repeat in Chrome to compare

Note: On first visit, the SW registers but may not intercept until the next navigation. Reload once after seeing "SW registered" to activate interception.

What This Page Does

A Service Worker (sw.js) intercepts every navigation request and modifies the HTML response — exactly like a network proxy (AdGuard) does during HTTP transit:

  • Injects <style id="sw-proxy-injected"> into <head>
  • The injected CSS hides .ad-placeholder elements and sets a CSS variable
  • On session restore, if the browser serves from cache without going through the SW, the injected style is missing

This directly simulates the real-world scenario where AdGuard injects CSS into 423down.com during HTTP transit, and Firefox's session restore loses those injections.
