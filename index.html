How to Test

The bug requires a network proxy (e.g. AdGuard) to manifest. Without a proxy, both browsers behave identically.

  1. Install and enable AdGuard (or any MITM content-filtering proxy)
  2. Open this page in Firefox over HTTPS
  3. Verify Probe 7 shows the proxy is active (styles in DOM match styles in fetch)
  4. Wait 3+ seconds for all delayed modifications to complete
  5. Test A — Full browser restart (primary test):
    Close the entire browser (Quit Firefox) → Reopen → Session restore activates the tab
    bfcache cannot survive browser restart — this forces actual session restore
  6. Test B — Ctrl+Shift+T (may use bfcache — less reliable):
    Close tab (Ctrl+W) → Restore (Ctrl+Shift+T)
    If Probe 6 shows bfcache, the page was just thawed from memory (normal)
  7. Check Probe 7 (the key test):
    • Red ❌ PROXY BYPASS = DOM has fewer styles than a fresh fetch → proxy modifications were lost
    • Green ✅ = DOM matches fresh fetch → proxy styles preserved
    Probes 1-3 detect DOM snapshot artifacts. Probes 4-6 are informational.
  8. Repeat in Chrome — Probe 7 should always show green (proxy styles preserved)

What This Page Tests

Core bug: When Firefox restores a session (browser restart or Ctrl+Shift+T), it serves the page from HTTP cache — the original server response, before any network proxy could modify it. This means:

  • AdGuard/proxy-injected CSS (added during HTTP transit) is lost → ads flash back into visibility
  • Proxy-injected scripts are lost → content filtering, userscripts, and cosmetic rules break
  • Any MITM modification is lost — the browser serves the pre-proxy cached response

This page emulates patterns from 423down.com (a confirmed reproducer with AdGuard) and uses fetch() to detect the proxy bypass.
