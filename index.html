    How to Reproduce

    This page must be served over HTTP/HTTPS (not file://). Use python3 -m http.server for local testing, or deploy to GitHub Pages.
    1. Open this page in Firefox (no extensions, clean profile recommended)
    2. Note the banner — it should say "Fresh load from server" (green)
    3. Optionally click "Inject <style>" to change the background to blue
    4. Close the tab (Ctrl+W / Cmd+W)
    5. Restore the tab (Ctrl+Shift+T / Cmd+Shift+T)
    6. Check the banner:
      • Red "SNAPSHOT RESTORE" — Firefox loaded the page from its internal DOM snapshot instead of re-fetching from the server. This is the bug.
      • Green "Fresh load" — the page was re-fetched from the server. This is correct behavior.
    7. Repeat the same steps in Chrome — it should always show green (fresh load)

    What This Proves

    When Firefox restores a tab via Ctrl+Shift+T or session restore (browser restart), it may reconstruct the page from an internal DOM snapshot rather than re-fetching from the server. This snapshot includes modifications made by JavaScript during the previous page load.

    This causes problems for:

    • Dynamic websites — server-side state changes (e.g. GitHub labels, comments) are not reflected because the page is never re-fetched
    • Network proxies — tools like content filters inject CSS/JS during transit; the snapshot bypasses the network entirely, so injections are lost
    • Cache-Control headers — no-store and no-cache are ignored because no HTTP request is made
