How to Test

  1. Open this page in Firefox over HTTP/HTTPS (no extensions, clean profile)
  2. Verify the banner says "Fresh load" (green) and all probes show ✅
  3. The red "SIMULATED AD" boxes should disappear within 1 second
  4. Wait 3 seconds for all delayed modifications to complete
  5. Test A: Close the tab (Ctrl+W) → Restore (Ctrl+Shift+T)
  6. Test B: Close the entire browser → Reopen → Let session restore activate the tab
  7. Check probes:
    • Red ❌ = snapshot/cache evidence (bug)
    • Green ✅ = fresh load (correct)
  8. Repeat in Chrome — should always show all green

What This Page Emulates

This page replicates the dynamic DOM modification patterns found on 423down.com, a real-world site where the Firefox session restore bug is visible with content filtering proxies:

  • Dynamic <style> injection on DOMContentLoaded — hides "ad" elements (emulates AdGuard CSS injection)

  • Delayed DOM modification via setTimeout(fn, 2000) — emulates ad script initialization
