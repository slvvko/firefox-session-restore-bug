How to Test

Important: Ctrl+Shift+T often uses bfcache (page frozen/thawed from memory) — that's normal browser behavior, not the bug. The real test is full browser restart.

  1. Open this page in Firefox over HTTP/HTTPS (no extensions, clean profile)
  2. Verify the banner says "Fresh load" (green)
  3. Wait 3+ seconds for all delayed modifications to complete
  4. Test A — Ctrl+Shift+T (may use bfcache):
    Close tab (Ctrl+W) → Restore (Ctrl+Shift+T)
    If Probe 6 shows "bfcache: YES", that's normal — page was just thawed from memory
  5. Test B — Full browser restart (the real test):
    Close the entire browser → Reopen → Enable "Restore previous session" → Check probes
    bfcache cannot survive browser restart — this forces actual session restore
  6. Check Probes 1-3 (primary detectors):
    • Red ❌ = DOM snapshot artifact detected (this is the bug)
    • Green ✅ = scripts ran on fresh HTML (correct)
    Probes 4-6 are informational — back_forward + transferSize=0 is normal.
  7. Repeat in Chrome — should always show all green for probes 1-3

What This Page Emulates

This page replicates the dynamic DOM modification patterns found on 423down.com, a real-world site where the Firefox session restore bug is visible with content filtering proxies:

  • Dynamic <style> injection on DOMContentLoaded — hides "ad" elements (emulates AdGuard CSS injection)
